# Audits

Structured findings against a closed scope. Each audit captures what
was checked + how it was checked + which items pass/fail at the audit
date. See
[`../operations/doc_conventions.md`](../operations/doc_conventions.md#type-audit)
for the canonical structure.

Audits can be either time-bound snapshots (`YYYY-MM-DD_topic-slug.md`)
or evergreen living docs that are updated as items resolve
(`topic-slug.md`). The audit doc's own `status` and `date` frontmatter
indicate which mode it operates in.
