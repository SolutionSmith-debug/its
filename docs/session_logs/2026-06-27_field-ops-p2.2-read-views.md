## Decisions made during session

- **A's first attempt was DISCARDED, not patched.** The Pit Wall Le Mans agent built A in a 16K context
  window (operator caught the misconfig — against the default-to-largest guidance) and it drifted hard:
  string-interpolated SQL (injection), wrong gate convention, bespoke localStorage/Bearer auth, broken
  keyset. Reverted the worktree to the clean Brief-0 baseline and re-dispatched the reset agent at 262K
  with a hard-anchored prompt quoting every contract it got wrong. The retry was clean. Rationale:
  the output was unsalvageable (every file structurally wrong), and the root cause (window size) was a
  config fix, not a capability gap.
- **Execution model per brief:** A and B via Le Mans + a CC verification pass; **C CC-led directly**
  (operator's call) because it was the most complex (5 legs, fan-out fix, F5 guard). Le Mans reliably
  gets the *structure* right on contained briefs but needs a real CC verify pass each time — B needed a
  substantial one (windowed-batch inner-column 500s, a worker↔frontend response-shape mismatch that
  would have broken the live page, invalid 403 gate tests, a full SPA-test rewrite). C, CC-led, came
  together with one type fix then green.
- **Design-doc schema bug found + fixed in the doc** (`cc-brief_p2.2-readviews.md`, ⚠️ SCHEMA CORRECTION
  block): the workflow-authored SQL assumed a `recorded_at` column that exists ONLY on
  `equipment_location`; `time_entries`/`inspections`/`equipment_logs` use `created_at`. Caused the 500s in
  A/B/C. Fix = alias `created_at AS recorded_at` (in the inner subquery for windowed batches). Also (C):
  jobs has `client_id`, not `client_name` → LEFT JOIN `clients`.
- **C's crew/open-tasks list batches windowed to ≤20/job IN SQL** (`ROW_NUMBER` + `rn <= NESTED_CAP`)
  rather than the brief's "fetch + cap-in-JS" — bounded O(page·cap), satisfies scalability checklist
  item 3, matches B's pattern. Chosen over the brief's literal SQL for defensiveness.
- **Bundled the cursor.ts hardening into A's PR** (one PR) rather than a separate shared-infra PR —
  operator's choice; it was a prerequisite for A's own security gate.
