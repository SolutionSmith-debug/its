## Cutover (operator-run, CC-guided) + two snags CC fixed live

Operator ran: migrations `0021`+`0022 --remote`; the token pair (`PORTAL_FIELDOPS_API_TOKEN` Worker
secret == `ITS_PORTAL_FIELDOPS_TOKEN` Keychain); Worker `npm run deploy`; loaded the `fieldops-sync`
plist; set `field_ops.fieldops_sync.sync_enabled=true`. Two snags surfaced and CC fixed both via the
Smartsheet MCP (write access, operator-authorized "make it correct"):

1. **Slice-4 was half-done.** The operator retyped `ITS_Active_Jobs."Job ID"` AUTO_NUMBER→TEXT but
   did NOT add the `Portal Job Key` column → the daemon's find-or-create errored (permanent
   SmartsheetValidationError → Review Queue), nothing mirrored. Root cause found by reading the live
   columns (column absent). **CC added the `Portal Job Key` TEXT column** (`add_columns`). Because
   the daemon leaves a failed job DIRTY (never mark-mirrored — verified in `_mirror_job`), a reload
   re-pulled and it mirrored on the next cycle. No re-create needed.
2. **Config-workstream resolution gap.** The daemon reads ALL its ITS_Config under
   `Workstream=field_ops`, including the shared `safety_reports.portal.worker_base_url` — so flipping
   `sync_enabled` alone would have fail-closed (`_resolve_credentials` → None → CRITICAL). **CC added
   both `field_ops` rows** (`sync_enabled=true` + a `worker_base_url` copy) via `add_rows` (`field_ops`
   isn't a Workstream picklist option → written free-text via `strict:false`). This left a duplicate
   base-URL row — a divergence hazard, tracked as a fast-follow (fix `_resolve_credentials` to read
   the shared key under `safety_reports`).

## Concurrency audit — SAFE (the operator's "will the daemons conflict?" answered with evidence)

A 4-agent workflow audited every shared write path across the 11-daemon fleet. Verdict: **SAFE**
(local-state), **SAFE_WITH_CAVEATS** (one low-risk hourly-picklist-sync self-overlap, idempotent).
Every shared write is guarded — per-daemon `fcntl` flocks (self-overlap), the shared
`heartbeat_row_ids.json` via `state_io` atomic-write + sidecar-lock, the host compile mutex
(weekly/progress generate), find-or-create idempotency on Smartsheet writes, the circuit breaker on
rate limits, and staggered intervals. Zero local-state bypasses (no raw `write_text` under `state/`).
The key reframe for the operator: the website (Cloudflare Worker + D1) is DECOUPLED from the Mac
daemons — a daemon hiccup delays back-end filing, it cannot take the site down; the real risk is
single-host availability (watchdog + UptimeRobot), not corruption.
