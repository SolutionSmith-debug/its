Restart dashboard (Class B · elevated · self-restart)

Restart the dashboard's own daemon so it picks up the code currently on ~/its — the one deliberate exception to "a service must not stop itself" (operator-authorized, DASH-12). Restart only: this never pulls or deploys anything. The page drops for a few seconds while launchd brings it back; reload afterwards and check /healthz. Elevated: re-enter the PIN + type restart-dashboard.

Circuit breaker (Class B · elevated)

Reset a stuck-OPEN circuit breaker to CLOSED (skip the cooldown). Read the current state in the circuit breaker panel first. Elevated: re-enter the PIN + type clear-breaker. (Stale state-locks are not clearable — a dead holder's flock auto-releases; see the runbook.)

Error log (Class B · elevated)

Mark resolved → then clear. Stamp Resolved At on open CRITICALs matching a Script and/or Error code (e.g. a retired daemon, or a fixed error class) — this makes them terminal, after which the clear button below can remove them. Only open CRITICALs are stamped; a filter is required (an unfiltered mass-resolve is refused — it would empty the "am I on fire" surface). Elevated: re-enter the PIN + type mark-resolved. Click preview first — it counts the matching open CRITICALs without writing, so you can check the blast radius (a Script alone sweeps every Error code under that daemon) before committing.

Clear terminal rows from ITS_Errors on demand — the operator-triggered version of the watchdog's automatic row-cap rotation (Check O), with no 90-day floor. Open CRITICALs are never deleted (they are the open CRITICALs panel's "am I on fire" surface); the clear/rotation audit trail is preserved. Optionally keep recent rows with an older-than day count (blank = every terminal row). Bounded to 4,600 rows per run — re-run to continue a large backlog. Elevated: re-enter the PIN + type clear-error-log.

Irreversible. Cleared rows are gone from the forensic log. Open CRITICALs and the clear/rotation audit trail survive; everything else terminal in scope is permanently removed.

Review queue (Class B · elevated)

Resolve PENDING ITS_Review_Queue rows matching a Workstream and/or Summary prefix — for clearing a stale recurring class (e.g. a removed sandbox job's compile-failure backlog) once its cause is fixed or moot. Rows move to REJECTED (noise / stale) or APPROVED (handled out-of-band) with a stamped date + note; nothing is deleted, and a filter is required. Click preview first — it counts the matches without writing. Elevated: re-enter the PIN + type resolve-review.

Secret rotation (Class C · write-only · elevated)

Values are write-only — never displayed or read back. Rotating a Worker bearer also writes its byte-equal Keychain mirror. The Box refresh token rotates only via the guided quiesce → setup_box_oauth.py → smoke flow (see the runbook).

Irreversible. A rotation takes effect immediately and overwrites the old secret in place — it cannot be undone from here. Confirm the new value is correct before you type the credential name to proceed.

Change operator PIN (Class C · elevated)

Change, not recovery. Developer-Operator credential self-service (Op Stds §44 rider). This requires your current PIN. A lost PIN is recovered only from the terminal — security add-generic-password -U -a "$USER" -s ITS_OPERATOR_PIN -w. New PIN: ≥ 8 chars, not all-digits, not blank, not one repeated character; write-only (never shown), so enter it twice; it takes effect immediately.

Build-time settings (Class D · hand-off)

Settings that take effect only by re-bundling/deploying the Worker (the terms library; today purchaser/tax too) are edited via the §50 config editor in the Safety Portal — it enqueues to config_requests and the config_actuator daemon does the privileged commit → CI → merge → deploy. This dashboard never deploys. Actuator faults surface there and escalate per docs/runbooks/config_actuator.md.
